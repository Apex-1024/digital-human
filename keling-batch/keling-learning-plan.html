2. 学习哲学

核心原则

这个项目是 learn-by-building 的绝佳案例。它不是单纯"读代码"，而是鼓励你理解每一步之后亲手重建：

  1. 从零开始 — 每个模块都可以独立重写，从最小可行版本开始
  2. 迭代叠加 — OCR 先做 RapidOCR 基线，再加 pix2tex 公式增强，最后加符号转中文
  3. 三级回退 — PPT 渲染（COM → LibreOffice → Pillow）、数字人（SadTalker → 静态头像）都是经典的回退设计模式
  4. 实战产出 — 每个阶段都有可运行的视频作为验证

4. 模块一：PPT 解析与渲染

4.1 三级回退：PPT → PNG

方案技术速度质量适用场景
方案 A（首选） PowerPoint COM (pywin32) ~2秒 ⭐⭐⭐ 完美 Windows，含公式
方案 B（回退） LibreOffice headless ~15分钟 ⭐⭐ 一般 Linux/Mac，纯文字
方案 C（兜底） Pillow 纯文本重绘 ~1秒 ⭐ 公式全丢 无 Office 环境
关键教训：方案 C 会覆盖方案 A 的高清 PNG，导致公式丢失。必须确保回退链上层的输出不被下层覆盖。

4.2 python-pptx 的局限

python-pptx 无法读取 MathType OLE 嵌入对象（progId="Equation.DSMT4"），导致含公式的段落被提取为残缺文本：

原始 PPT 段落:  ∫_T1^T2 v(t)dt = s(T2) - s(T1)

python-pptx 提取: "  2 v(t)dt  s(T )  s(T )."  ← 公式全丢！

这引出了模块二的核心需求：OCR 补充公式识别。

5. 模块二：OCR 数学公式识别

5.1 双引擎架构

引擎用途依赖稳定性
RapidOCR（首选） 文字检测 + 文本识别 onnxruntime（纯 Python） ⭐⭐⭐ 极稳定
PaddleOCR（回退） 同上 paddlepaddle（C++ 编译） ⭐⭐ 版本兼容问题多
pix2tex（增强） 公式图片 → LaTeX ViT + Transformer 模型 ⭐⭐ 模型大(97MB)，下载慢

5.3 公式判定：_looks_like_formula()

_MATH_HINT_CHARS = set("=±÷×∑∫√π∞αβγδθλμνΦφΩω≤≥≠≈→←↔∈∉∀∃∝·²³")

def _looks_like_formula(text: str) -> bool:
    if any(c in _MATH_HINT_CHARS for c in text):
        return True
    if "\\" in text or "^" in text or "_" in text:
        return True
    digits = sum(1 for c in text if c.isdigit())
    if "=" in text and digits >= 1:
        return True
    return False

5.4 碎片清理逻辑

python-pptx 提取的公式碎片（如 "T1"、"v(t)dt"、"∫"）需被清理。核心策略：

  1. 不含中文 的 bullet 视为疑似碎片
  2. 单字符/纯标点(≤2字符) → 直接删除
  3. 短碎片(≤6字符) → 检查 OCR 是否有更完整版本包含它
  4. 较长残缺公式(≤10有效字符) → 字符重合≥50% 且 OCR 更长则删除

6. 模块三：符号转中文口播

6.1 双函数体系

函数输入输出示例
latex_to_chinese() LaTeX 公式 中文口播读法 \int_a^b f(x)dx → "从 a 到 b 的积分 f(x) 微元 d x"
math_symbols_to_chinese() 普通文本含符号 中文口播读法 v = v(t), v(t) ≥ 0 → "v 等于 v(t), v(t) 大于等于 0"

6.2 MathType 私有 Unicode 字符映射

这是本模块最大的隐藏坑：

python-pptx 提取 MathType 公式时，Symbol 字体字符被映射到 Private Use Area (U+F000+)，而不是标准 Unicode 数学符号：

mathtype_map = {
    "\uf03d": "=",    # equal (Symbol 0x3D)
    "\uf0b3": "≥",    # greaterequal (Symbol 0xB3)
    "\uf02d": "-",    # minus (Symbol 0x2D)
    "\uf0b4": "≠",    # notequal (Symbol 0xB4)
    # ... 40+ 映射
}

6.3 latex_to_chinese 覆盖的 LaTeX 命令

  • 希腊字母：\alpha→阿尔法, \Phi→弗爱, 等 30+ 个
  • 积分：\int_a^b → "从 a 到 b 的积分"
  • 求和：\sum_{i=1}^n → "i 到 n 求和"
  • 分数：\frac{a}{b} → "b 分之 a"
  • 根号：\sqrt[n]{x} → "n 次根号下 x"
  • 上下标：a^{bc} → "a 的 bc 次方"；单字符下标直接拼接（T_{2}→T2）
  • 极限：\lim_{x\to a} → "极限 x 趋近于 a"

6.4 math_symbols_to_chinese 覆盖的符号

  • 关系符：=→等于, ≥→大于等于, ≤→小于等于, ≠→不等于, ≈→约等于
  • 算术：+→加, -→减, ×→乘以, ÷→除以
  • 区间：[T1,T2]→T1 到 T2
  • 积分：∫→积分
  • 导数：s'(t)→s 撇 (t)
  • 下标：仅多字符下标才读"下标"（T12→T 下标 12），单字符直接拼接（T1→T1）

7. 模块四：TTS 语音合成

edge-tts 7.2.8+

必须使用 edge-tts ≥ 7.2.8，旧版本使用已废弃的 Sec-MS-GEC token 算法会返回 403。

可用音色：zh-CN-XiaoxiaoNeural（女声）、zh-CN-YunxiNeural（男声）等 7 种中文音色。

def tts_synthesize(text: str, voice: str, out_path: str) -> float:
    """用 edge-tts 合成中文语音，返回音频时长（秒）。失败时回退到 pyttsx3。"""
    import edge_tts
    communicate = edge_tts.Communicate(text, voice)
    asyncio.run(communicate.save(out_path))
    # 用 ffprobe 获取实际时长
    return get_audio_duration(out_path)

8.2 左右分栏布局（方案 B）

side_w = 320   # 侧栏 25%
avatar_w = 224 # 头像 70% 侧栏宽度
avatar_x = 960 + (320-224)/2 = 1008  # 头像左侧 1008 > PPT 右侧 960 ✓

8.3 FFmpeg filter_complex 结构

filter_complex = (
    # 1. PPT 缩放到主区域，保持比例，加黑边
    "[0:v]scale=960:540:force_original_aspect_ratio=decrease,"
    "pad=960:540:(ow-iw)/2:(oh-ih)/2:black,setsar=1,fps=25[bg1];"

    # 2. pad 到整画布 1280×720，右侧填深蓝色
    "[bg1]pad=1280:720:0:90:0x1a1f3a[bg];"

    # 3. 头像缩放
    "[1:v]scale=224:224,fps=25[avt];"

    # 4. 头像叠加到侧栏居中
    "[bg][avt]overlay=1008:248[v]"
)

11. 实战避坑经验

🐛 MathType 私有 Unicode

python-pptx 提取的 = 是 \uf03d，≥ 是 \uf0b3。没有映射表的话，这些符号在 TTS 里会静音或被读成乱码。

🐛 pix2tex 把 v 识别为 ν

微积分教材中速度 v 被 pix2tex 识别成希腊字母 \nu（纽）。需要手动修正：latex.replace(r"\nu", "v")。

🐛 PaddleOCR 版本地狱

PaddleOCR 3.x 强制下载 UVDoc 模型，2.x 与 paddlepaddle 3.x 不兼容，numpy 2.x 与 paddlex 冲突。最终弃用 paddle，改用 RapidOCR。

🐛 去重坐标抖动

RapidOCR 返回的重复项 y 坐标有 1-2px 差异，导致 round(y/10) 量化桶失效。改用 int(y // 20) 20px 桶。

🐛 Pillow 覆盖高清图

pywin32 未安装时，Pillow 简化渲染会覆盖 PowerPoint COM 生成的高清 PNG，公式全丢。必须在回退链中保护上层输出。

🐛 ffmpeg 必须用完整版

系统精简版 ffmpeg 缺少 lavfi/aac 编码器，必须用 imageio-ffmpeg 提供的完整静态二进制（31MB，含 ffmpeg-win-x86_64-v7.1.exe）。
